# Strategy: MAXIMUM AI VISIBILITY
# All public content is accessible to all crawlers -- search, training, and
# traditional. Private paths (auth, API, community, dev) blocked consistently.
#
# AI search crawlers (ChatGPT-User, Perplexity-User, etc.) fetch content in
# real time when users ask questions -- allowing them = appearing in AI answers.
#
# AI training crawlers (GPTBot, ClaudeBot, etc.) build future model knowledge.
# For a young site building authority, training inclusion = future visibility.
# To block training later, change their Allow: / to Disallow: /
# =============================================================================

# --- Content Signals (contentsignals.org / draft-romm-aipref-contentsignals) --
# Per the IETF draft + Lighthouse robots.txt validator, Content-Signal is
# delivered as an HTTP response header. We ship it on every response via
# functions/_middleware.js -> withSecurityHeaders() (header value:
# `ai-train=yes, search=yes, ai-input=yes`). Keeping the directive here too
# trips Lighthouse "robots.txt is not valid" because the validator only
# understands the original 1994 spec (User-agent / Allow / Disallow /
# Sitemap). Removed from this file 2026-05-15.
